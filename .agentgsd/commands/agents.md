Create an AGENTS.md file in the project root following best practices. The file should be maximum 250 lines and include:

## Structure (follow this order):

1. **Header** - Brief description of the project (1-2 sentences)

2. **Tech Stack** - List of technologies, frameworks, languages used

3. **Project Structure** - Key directories and their purpose (use tree format if helpful)

4. **Coding Standards** - Linting rules, formatting preferences, naming conventions

5. **Common Tasks** - How to perform common operations:
   - Running tests
   - Building the project
   - Running linters/formatters
   - Database migrations

6. **Important Files** - Key configuration files agents should know about

7. **Constraints** - What NOT to do, security considerations, deprecated patterns

8. **Testing** - Test file locations, how to run specific tests

Make it concise and actionable. Use bullet points and code examples where helpful.
